## Verdict

Spec is buildable but **phase 1 is mis-sized**. It reads as "extract a package and rename some classes" — it is actually a full refactor of `apps/transline49-web/` (604 `tl-*` occurrences across 19 files, 144 unique tokens, plus `data-tlgrid*` data attributes and `@keyframes tl-fadein` / `tl-marquee` rules) followed by a workspace-config rewire that touches root `package.json`, root eslint, both apps' `next.config.mjs`, and TL49's `lint` script. Several sub-items in §1 (bundle-budget CI, build-time PDF/href checks, the carousel) are presented as if they're already-solved patterns when no one has written them. Cut what doesn't earn its keep, sequence the foundation work properly, and Miller phase 2 onward becomes the cleanly parallel work the spec advertises.

## Where engineering will get stuck

1. **The `tl-*` → `wo-*` rename is not a sed.** 144 unique class tokens, BEM-style siblings (`tl-card--dark`, `tl-faq__btn`, `tl-mobile-nav__lbl`), CSS keyframes (`@keyframes tl-fadein`, `tl-marquee`, `tlpfx-truck-in`), and the `data-tlgrid` attribute family at `app/globals.css:1586+`. A naive `tl-` find/replace also hits the **tweaks panel CSS** (`.twk-*` is fine, but `controls.jsx` and `styles.js` need to be excluded from the pattern) and risks rewriting unrelated substrings inside Unsplash URLs or copy strings. The engineer will write one sed, run `npm run dev`, find the page un-styled, then bisect for two hours. **Spec change:** in §1.2 specify the migration as an explicit `git mv`-style codemod with a token allowlist (the 144 classes + `tlpfx-`, `tlgrid`, `tl-fadein`, `tl-marquee` keyframes), require running `npm run lint && npm run build` clean before commit, and call out that `apps/transline49-web/components/tweaks/` is excluded. Also rename `tlpfx-truck-in` and friends explicitly — the spec only mentions `tl-`.

2. **Workspace root currently has no eslint config; per-app config uses `eslint-config-next/core-web-vitals`.** `apps/transline49-web/eslint.config.mjs` imports `eslint-config-next` from the app's own `devDependencies`. Hoisting to `<repo root>/eslint.config.mjs` means: (a) move `eslint` + `eslint-config-next` to root `devDependencies`, (b) delete the per-app config (or stub it to re-export root), (c) change `apps/transline49-web/package.json`'s `"lint": "eslint ."` to `"lint": "eslint . --config ../../eslint.config.mjs"` *or* drop per-app lint scripts entirely. The spec says "per-app `lint` scripts shell to it" without specifying which. Engineer will pick wrong and ship two configs. **Spec change:** in §1.2 state explicitly: "Per-app `lint` script is removed; only `npm run lint` at root works. App-level IDE integration uses root config via flat-config auto-discovery."

3. **`@white-owl/brand` install ordering.** Root `package.json` globs `apps/*` but the new package lives at `apps/shared/brand/`. `npm install` from root will resolve the workspace once the package.json exists, but `apps/transline49-web/package.json` doesn't currently list `@white-owl/brand` as a dep — symlink resolution succeeds only if the consumer declares the workspace dep. Engineer will create the package, run `npm install`, see no symlink in `apps/transline49-web/node_modules/@white-owl/`, and chase phantom Next 16 errors. **Spec change:** §1.2 should call out the explicit sequence: (a) create `apps/shared/brand/package.json`, (b) add `"@white-owl/brand": "*"` to both `apps/transline49-web/package.json` and `apps/miller-web/package.json` `dependencies`, (c) `npm install` at root, (d) verify with `ls apps/transline49-web/node_modules/@white-owl/brand`. Also: `apps/*` glob is **not recursive** — it will pick up `apps/shared/` as a workspace but **not** `apps/shared/brand/`. Workspaces field must change to `["apps/*", "apps/shared/*"]` or the package needs to be at `apps/brand/`.

4. **Bundle-budget CI gate doesn't exist yet.** `@next/bundle-analyzer` produces an interactive HTML treemap, not per-route byte counts. Next 16 with App Router writes per-route manifests to `.next/app-build-manifest.json` plus chunk maps in `.next/static/chunks/`, but parsing those into "shipped bytes per route" requires walking the chunk graph (entry + shared + dynamic imports), gzipping, summing. The "custom script" the spec hand-waves is ~150 lines of code, none of it written. Engineer will spend a day on it before any Miller page lands. **Spec change:** either (a) demote bundle budget to a phase-4 manual check using `npm run build`'s console output (Next prints first-load JS per route — good enough), or (b) lift the script into a real sub-task with its file path (`scripts/check-bundle-budgets.mjs`) and a worked example using `.next/app-build-manifest.json`.

5. **Build-time PDF / `href="#"` check is also unwritten.** §6 says "Build-time check fails if any `href="#"` placeholder ships." Next has no hook for that. Implementing it means a custom post-build script that walks `.next/server/app/**/*.html`, parses for `href="#"`, and exits non-zero. Engineer hits it during phase 3 when half the cert PDFs are stubs, fails CI, and turns the check off. **Spec change:** replace with an eslint rule (`no-restricted-syntax` matching JSX `href="#"`) plus a CI grep on `lib/certs.js` for `href: "#"`. Both are 5-minute additions. Or accept that placeholders ship until phase 4 and gate the check there.

6. **The carousel "hold 3s before swap, pure CSS, respects `prefers-reduced-motion`" needs more spec.** §7 says "Carousel holds frame 1 for ≥ 3 s before first swap" and §3.1 mentions a "CSS-keyframed slideshow." With pure CSS, the typical pattern is `animation: slideshow 12s infinite; animation-delay: 0s` on three stacked layers with offset opacity keyframes. Holding frame 1 for exactly 3s while keeping LCP on frame 1 means frame 1 must have `opacity: 1` from `0%` to `25%` of the cycle (3s of a 12s loop). Reduced-motion freeze means `animation: none` AND ensuring frames 2/3 don't render at all (otherwise they're stacked on top with `opacity: 0` initially but consume layout). `prefers-reduced-motion: reduce { .frame-2, .frame-3 { display: none } }`. None of this is in the spec; engineer reinvents it. **Spec change:** drop a 12-line CSS sketch into §7 or §4.5.

7. **`lib/content/*` as a layer.** Listed: `brand.js, home.js, about-*.js (5), service-*.js (10 + landing), treatment-facility.js, winnipeg.js, careers-*.js, contact.js, process-oxidizers.js, case-studies.js` ≈ **23 files** for a static marketing site. If content is just strings + arrays of bullets, this is fine — but the moment a page wants a `<strong>` mid-sentence, content modules sprout JSX, become components-in-disguise, and lose the "templates render plain data" property the spec promises. TL49 today does **not** have a `lib/content/` layer; copy lives in JSX. Engineer will either follow the spec rigidly and end up with `lead: <>Over 25 <strong>years</strong>…</>` (JSX in `.js`, eslint complaints) or quietly skip it. **Spec change:** explicitly scope `lib/content/*` to plain-text strings + arrays only; any inline emphasis stays in JSX templates. Or limit it to `brand.js` (the cross-page-shared strings) and let each page colocate its own copy.

## Simplifications to consider

- **Drop the `Processes` top-nav slot** (§10 already raises this). One direct-link nav item to one page is dead weight; fold into Services dropdown until a second process page exists.
- **Drop `motif="cert-tick"`** (§4.2). The eyebrow rule is decorative; just hide the parallel-49 motif on Miller via a `data-brand="miller"` selector. No `motif` prop, no DOM-size-matching gymnastics, one less component API.
- **Drop the `namespace` factory on `SiteTweaksProvider`** (§1.2). Tweaks ship dev-only (§4.3); localStorage key collision between two `localhost:3000` dev sessions for different apps is a 10-second annoyance, not a real bug. A hard-coded key per app (`tweaks:tl49` literal in tl49's provider mount, `tweaks:miller` in miller's) is simpler than a factory.
- **Drop `FamilyOfCompanies` from phase 1.** It requires a TL49 footer edit (cross-app coupling) and a "current site is bolded" prop. The footer link list already lists "TransLine49°/Miller Waste Systems"-style sister brands in plenty of competitor sites — phase 4 polish, not foundation.
- **Cut `StatsBand` entirely until real numbers exist.** Rendering "Satisfied clients: data unavailable" to a screen reader is worse than not rendering the section at all. The "no invented stats" non-goal is already in the spec; honour it by deleting the section, not by shipping a placeholder.

## Phase 1 ordering — MUST land before any Miller page

In order, with verification gate per step:

1. **TL49 codemod + rename audit.** Build allowlist of 144 `tl-*` class tokens + `data-tlgrid*` + `tl-fadein` / `tl-marquee` / `tlpfx-*` keyframes. Single commit. Gate: `cd apps/transline49-web && npm run lint && npm run build` clean. Visual diff against deployed site (one engineer, 30 min) — no spacing, no animation, no nav regressed.
2. **Workspace layout fix.** Either move package to `apps/brand/` or change workspaces glob to `["apps/*", "apps/shared/*"]`. Decide before step 3 because path is referenced 17 times in spec.
3. **Create `apps/shared/brand/` (or `apps/brand/`) with extracted CSS + components only.** No tweaks yet. Add `"@white-owl/brand": "*"` to TL49's `dependencies`. Add `transpilePackages: ["@white-owl/brand"]` to TL49's `next.config.mjs`. Move CSS import in TL49's `app/layout.jsx`. Gate: TL49 still builds, still visually matches.
4. **Hoist eslint.** Move deps to root, write root `eslint.config.mjs`, delete per-app config, delete per-app lint scripts. Gate: `npm run lint` at root passes.
5. **Move tweaks into shared package, gate with `NODE_ENV !== "production"`.** Gate: production build of TL49 has zero `twk-*` strings in client chunks (grep `.next/static/chunks/*.js`).
6. **Scaffold `apps/miller-web/` skeleton** — `package.json`, `next.config.mjs` with `transpilePackages`, `app/layout.jsx`, empty home page. Gate: `npm run dev:miller` boots, root `npm run build` builds both apps.

Anything beyond step 6 is phase 2. The bundle-budget script, build-time href check, and `FamilyOfCompanies` cross-coupling slip to phase 4 where they belong.
